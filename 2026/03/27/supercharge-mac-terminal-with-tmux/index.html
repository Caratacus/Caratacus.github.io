Mac 终端不够用？tmux 让你的终端战斗力翻倍

标签： Terminal

作为 Mac 用户，自带终端或 iTerm2 已经能满足日常 ls、cd、git 等基础操作，甚至支持多标签和简单分屏。但如果你遇到过这些问题：

  • 手滑关掉终端，跑了半小时的编译/下载/爬虫直接中断
  • SSH 连服务器时网络波动，远程任务瞬间”蒸发”
  • 想同时看代码、跑服务、查日志，多标签切换得眼花缭乱
  • 自带分屏不够灵活，无法自由调整布局、保留工作状态

那你一定需要 tmux——一款终端里的”窗口管理器 + 后台保活神器”，堪称开发者的终端效率利器。

什么是 tmux ？

tmux 本质是 “终端内的独立桌面”：它能在单个终端窗口中创建多个会话、窗格，并且最重要的是——所有会话和任务会在后台持续运行，哪怕你关掉终端、断网、锁屏，再次连接后依然能回到之前的工作状态，不会丢失任何进度。

简单说：tmux 解决了”终端一关，任务全没”的核心痛点，同时把分屏、多任务管理做到了极致。

tmux 核心功能

后台保活：任务永不中断

这是 tmux 最无可替代的优势：

  • 自带终端：关窗口 = 终止所有程序；断 SSH = 远程任务直接挂掉
  • tmux：关终端只是”隐藏”会话，任务在后台继续运行，下次用 tmux attach 就能无缝衔接

适用场景：跑编译、大数据下载、模型训练、长期爬虫、远程服务器任务

灵活分屏：一个终端 = 多个窗口

自带终端的分屏功能有限，tmux 支持：

  • 无限切分：上下分屏、左右分屏、混合分屏，想怎么切就怎么切
  • 自由调整：拖拽窗格大小、关闭单个窗格、快速切换焦点
  • 布局复用：保存常用布局，下次直接调用，不用重复设置

适用场景：同时写代码 + 跑服务 + 看日志 + 查文档，不用来回切换标签

SSH 远程开发必备

只要是经常连服务器的开发者，tmux 几乎是标配：

  • 解决痛点：网络波动、本地关机导致的 SSH 断开，远程任务不会丢
  • 用法：在服务器上启动 tmux 会话，本地断开后再连，用 tmux a 就能回到之前的工作界面

多会话管理：隔离不同工作场景

你可以同时创建多个独立会话，互不干扰：

  • 会话 1：开发项目 A（代码 + 调试）
  • 会话 2：运行项目 B 服务
  • 会话 3：处理服务器运维

随时切换会话，不用开多个终端窗口，桌面更整洁。

优化终端体验：复制粘贴 + 滚动更顺手

自带终端的复制粘贴、滚动历史在复杂场景下很别扭，tmux 配合简单配置就能实现：

  • 鼠标滚动查看历史输出
  • 跨窗格复制粘贴
  • 自定义快捷键，操作更高效

四、tmux 基础用法：常用命令全收录（满足 90% 场景）

整理了高频使用的核心命令，按”会话管理 + 窗格操作 + 窗口操作”分类，方便记忆：

操作类型 具体操作 命令 / 快捷键
会话管理 新建默认会话 终端输入 tmux
新建命名会话（推荐） 终端输入 tmux new-session -s name（简化写法：tmux new -s name）
列出所有后台会话 终端输入 tmux list-sessions（简化写法：tmux ls）
连接最近会话 终端输入 tmux attach（简化写法：tmux a）
连接指定命名会话 终端输入 tmux attach -t name（简化写法：tmux a -t name）
退出会话（保留后台） 会话内按 Ctrl+B → 再按 D（detach）
强制关闭指定会话 终端输入 tmux kill-session -t name
窗格操作 左右分屏（创建垂直窗格） 会话内按 Ctrl+B → 再按 %（英文百分号）
上下分屏（创建水平窗格） 会话内按 Ctrl+B → 再按 "（英文双引号）
切换窗格焦点 会话内按 Ctrl+B → 再按方向键（上下左右）
关闭当前窗格 会话内按 Ctrl+B → 再按 X → 回车确认
窗口操作 新建终端窗口（会话内多窗口） 会话内按 Ctrl+B → 再按 C（create）
切换会话/窗口（可视化选择） 会话内按 Ctrl+B → 再按 W（window/session），按方向键选择后回车

备注：

  1. 命名会话建议用项目名/用途命名（如 tmux new -s projectA），方便后续通过 tmux ls 快速识别
  2. 所有简化写法和完整写法功能一致，推荐记简化版（更高效）
  3. 按 Ctrl+B 时是”先按住 Ctrl+B 再松开，再按后续按键”，不是同时按

五、结语

tmux 的核心价值在于解耦终端会话与终端窗口的生命周期。这个看似简单的机制，从根本上解决了”关终端 = 杀进程”的痛点，尤其是对于 SSH 远程开发场景，这种解耦是刚需而非锦上添花。

如果你也有类似场景，那就赶紧安装起来吧！
